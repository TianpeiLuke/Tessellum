# Eufriction

## Definition

**Eufriction** is a term coined by Bob Doto (circa 2022) to describe *beneficial friction* in a Zettelkasten workflow — difficulty that is productive rather than obstructive. The concept specifically applies to the folgezettel numbering system, where the requirement to place each new note in a specific sequential position forces the practitioner to think carefully about where an idea belongs relative to existing ideas. This placement decision creates friction, but that friction generates deeper engagement with the material. The prefix "eu-" (Greek: good, well) parallels "eustress" (beneficial stress).

## Key Properties

- **Friction as feature, not bug**: The difficulty of finding the right placement for a note is itself a thinking exercise
- **Folgezettel-specific**: Most directly applies to sequential numbering systems where notes must be placed in a hierarchy
- **Forces comparison**: Placing a note requires comparing it to neighbours — "Is this idea more like 3a or 3b?" — which deepens understanding
- **Parallels desirable difficulties**: Bjork's learning science concept that certain obstacles enhance long-term retention and understanding
- **Contrasts with frictionless systems**: Tag-based or flat-file systems eliminate placement decisions, which removes this particular thinking opportunity
- **Not all friction is eufriction**: Poorly designed workflows create unproductive friction; eufriction specifically describes friction that produces better thinking
- **Applies beyond folgezettel**: Any system design choice that makes the user think harder about relationships (not logistics) can produce eufriction
