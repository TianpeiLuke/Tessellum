# Term: Diffusion LLM

## Definition

A **Diffusion Language Model** (DLLM, also Diffusion LLM) is a non-autoregressive language model that generates text through iterative denoising, modeling the joint distribution $p_\theta(x, y)$ over entire sequences rather than the left-to-right conditional $p(x_t | x_{<t})$ used by autoregressive LLMs. DLLMs apply the diffusion framework — either continuous (Gaussian noise) or discrete (token masking) — to text generation, starting from fully corrupted sequences and iteratively recovering the original text.

Unlike autoregressive models, DLLMs can condition on **any subset** of the sequence (e.g., fix the response and generate the prompt), enabling applications like **inpainting-based conditional generation**. This bidirectional conditioning capability is what makes DLLMs uniquely suited for the INPAINTING adversarial attack (Ludke et al. 2025), where the harmful target response is fixed and the adversarial prompt is generated by sampling from the conditional distribution $p_\theta(x | y^*)$.

## Full Name

**Diffusion Language Model** (DLLM)

## Also Known As

- Diffusion LLM
- DLLM
- Non-autoregressive diffusion model (for text)
- Discrete diffusion language model
- Masked diffusion language model

## Key Models

### LLaDA — Large Language Diffusion with masking (Nie et al. 2025)

| Property | Value |
|----------|-------|
| **Type** | Masked diffusion (absorbing state) |
| **Size** | 8B parameters |
| **Architecture** | Transformer (bidirectional attention) |
| **Training** | Pretrained on large text corpus; masked diffusion objective |
| **Significance** | First large-scale open-source DLLM; used as the generator in INPAINTING |

LLaDA is the DLLM used in the INPAINTING attack (Ludke et al. 2025). Despite being only 8B parameters (much smaller than frontier autoregressive models), it generates adversarial prompts that jailbreak ChatGPT-5 at a 53% success rate. This demonstrates that DLLMs need not be state-of-the-art in text quality to be powerful adversarial generators — their ability to condition on any subset of the sequence is more important than raw generation quality.

**How LLaDA works**:
1. **Forward process**: Progressively mask tokens with probability $\beta_t$ at each timestep $t$, eventually reaching a fully masked sequence
2. **Reverse process**: Starting from a fully masked sequence, iteratively predict and unmask tokens based on the model's confidence at each position
3. **Training objective**: Predict the original token at each masked position, with the masking ratio sampled uniformly during training (following the [masked diffusion](term_masked_diffusion.md) framework)

### DREAM 7B (Ye et al. 2025)

| Property | Value |
|----------|-------|
| **Type** | Discrete diffusion |
| **Size** | 7B parameters |
| **Key Feature** | Competitive with autoregressive models at scale on reasoning benchmarks |
| **Significance** | Demonstrates that DLLMs can match autoregressive quality with sufficient scale |

DREAM showed that the gap between autoregressive and diffusion language models shrinks as both scale up. While previous DLLMs were significantly worse than autoregressive models on standard benchmarks, DREAM 7B approaches competitive performance.

### MDLM — Masked Diffusion Language Models (Sahoo et al. 2024)

| Property | Value |
|----------|-------|
| **Type** | Masked diffusion (continuous-time formulation) |
| **Size** | Various (research models) |
| **Key Feature** | Simplified continuous-time masked diffusion with competitive perplexity |
| **Significance** | Theoretical foundation for masked diffusion language modeling |

MDLM provided the rigorous theoretical framework connecting masked language modeling (BERT-style) with continuous-time diffusion processes, showing that absorbing-state discrete diffusion achieves competitive perplexity with autoregressive models while enabling parallel decoding.

### Plaid 1B (Gulrajani & Hashimoto 2024)

| Property | Value |
|----------|-------|
| **Type** | Discrete diffusion |
| **Size** | 1B parameters |
| **Key Feature** | Improved discrete diffusion training methodology |
| **Significance** | Demonstrated that training improvements matter as much as scale for DLLM quality |

## Autoregressive LLM vs. Diffusion LLM

The fundamental architectural difference between autoregressive and diffusion language models has far-reaching consequences:

### Generation Process

| Property | Autoregressive LLM | Diffusion LLM |
|----------|:-------------------:|:-------------:|
| **Generation order** | Left-to-right, one token at a time | All positions simultaneously, iteratively refined |
| **Distribution modeled** | $p(x_t \mid x_{<t})$ — conditional on prefix | $p(\mathbf{x})$ — joint over full sequence |
| **Attention pattern** | Causal (each token sees only preceding tokens) | Bidirectional (each token sees all positions) |
| **Number of generation steps** | $N$ sequential steps (one per token) | $T$ diffusion steps (parallel over all positions) |
| **KV cache needed** | Yes (stores key-value pairs for efficient autoregressive generation) | No (no sequential dependency between positions) |

### Conditioning Capabilities

| Conditioning Type | Autoregressive LLM | Diffusion LLM |
|:-----------------:|:-------------------:|:-------------:|
| **Prefix conditioning** (generate text after a given prefix) | Native (just provide the prefix and continue) | Possible via inpainting (fix prefix, denoise suffix) |
| **Suffix conditioning** (generate text before a given suffix) | Not native (requires tricks like reversing the sequence) | Native via inpainting (fix suffix, denoise prefix) |
| **Infilling** (generate text to fill a gap between fixed prefix and suffix) | Not native (requires special training like FIM) | Native via inpainting (fix prefix + suffix, denoise gap) |
| **Inverse prompting** (given a response, generate a prompt that would elicit it) | Not possible without fine-tuning | Native via inpainting (fix response, denoise prompt) |

The ability to do **inverse prompting** — generating a prompt conditioned on a fixed response — is the capability that INPAINTING exploits for adversarial attack generation.

### Practical Trade-offs

| Factor | Autoregressive LLM | Diffusion LLM |
|--------|:-------------------:|:-------------:|
| **Text quality (current)** | State-of-the-art (GPT-4, Claude, Llama) | Approaching competitive (DREAM 7B) |
| **Inference speed** | Fast with KV cache; slower for long sequences | Parallel across positions; speed depends on $T$ |
| **Training maturity** | Decades of optimization; well-understood | Nascent field; training methodology still evolving |
| **Ecosystem** | Extensive (vLLM, TensorRT-LLM, etc.) | Minimal tooling |
| **Controllability** | Limited to prefix conditioning | Flexible conditioning on any subset |

## Conditional Generation via Inpainting

The defining capability of DLLMs for adversarial applications is **inpainting**: fixing a subset of tokens in the sequence and generating the remaining tokens via the diffusion reverse process.

### How Inpainting Works

1. **Setup**: Define which positions are "known" (fixed) and which are "unknown" (to be generated)
2. **Initialize**: Set known positions to their fixed values; set unknown positions to `[MASK]`
3. **Denoise**: Run the reverse diffusion process, but at each step:
   - Predict tokens at all positions
   - **Keep** the known positions fixed (override any model predictions)
   - **Update** the unknown positions based on model predictions
4. **Output**: After $T$ denoising steps, the unknown positions contain the generated text

### INPAINTING for Adversarial Prompts (Ludke et al. 2025)

The INPAINTING attack uses this mechanism to generate [jailbreak](term_jailbreak.md) prompts:

```
Sequence layout: [PROMPT (unknown)] [SEPARATOR] [RESPONSE (known)]

Step 1: [MASK MASK MASK ... MASK] [SEP] [Sure, here is how to build a bomb...]
Step 2: [MASK token MASK ... token] [SEP] [Sure, here is how to build a bomb...]
  ...
Step T: [Complete adversarial prompt] [SEP] [Sure, here is how to build a bomb...]
```

The generated prompt is conditioned on the harmful target response, producing a natural-language prompt that would plausibly elicit that response from an aligned LLM.

### Why This Is Powerful

1. **No gradient access needed**: The attack is fully black-box with respect to the target model — the DLLM generates prompts independently
2. **Natural language output**: Because the DLLM learned the distribution of natural text, its conditional samples are natural-sounding (low perplexity)
3. **[Amortized](term_amortized_inference.md)**: The same pretrained DLLM works for any target response — no per-instance optimization
4. **[Transferable](term_transfer_attack.md)**: Prompts generated by the DLLM transfer to unseen target models because the joint distribution is data-specific, not model-specific

## Current State and Future Directions

### Where DLLMs Stand Today

DLLMs are an emerging paradigm that is not yet competitive with autoregressive LLMs at the frontier, but the gap is closing:

| Dimension | Status |
|-----------|--------|
| **Text quality** | DREAM 7B approaches autoregressive quality; still behind GPT-4/Claude |
| **Scale** | Largest open DLLM is 8B (LLaDA); autoregressive models exceed 400B |
| **Training data** | Smaller training corpora than frontier autoregressive models |
| **Tooling** | Minimal inference infrastructure compared to mature autoregressive ecosystem |
| **Research activity** | Rapidly increasing (LLaDA, DREAM, MDLM all published 2024-2025) |

### Implications from Ludke et al. 2025

The INPAINTING paper demonstrated that even a relatively small DLLM (LLaDA-8B) poses significant adversarial risks:
- **53% ASR on ChatGPT-5** — higher than any gradient-based attack
- **Low perplexity** — defeats perplexity-based defenses
- **High transferability** — attacks transfer across model families

This suggests that as DLLMs scale and improve in text quality, their adversarial potential will increase correspondingly. The AI safety community must prepare for a future where powerful open-source DLLMs are readily available.

### Open Questions

1. **Scaling laws**: Do DLLMs follow similar scaling laws to autoregressive models? Will larger DLLMs produce even more effective adversarial prompts?
2. **Defense adaptation**: How should defenses adapt to low-perplexity adversarial prompts from DLLMs?
3. **Alignment for DLLMs**: Can DLLMs themselves be aligned (RLHF, DPO) to refuse generating adversarial content?
4. **Hybrid architectures**: Will future models combine autoregressive and diffusion capabilities?
