# Term: ReAct (Reasoning + Acting)

## Definition

**ReAct (Reasoning + Acting)** is a prompting framework that synergizes verbal reasoning traces and task-specific actions within large language models by interleaving them in a Thought-Action-Observation loop. Introduced by Yao et al. (2023, ICLR -- notable top 5%), ReAct prompts an LLM to alternate between (1) generating free-form **reasoning traces** (Thoughts) that decompose goals, track progress, and handle exceptions, and (2) emitting **actions** that interact with external environments (e.g., Wikipedia API, web browsers, embodied simulators) to gather information or change state. The environment returns an **Observation** after each action, which feeds back into subsequent reasoning. This interleaving enables two complementary synergies: *reason to act* (reasoning informs which action to take next) and *act to reason* (observations from actions ground the reasoning in external facts, reducing hallucination). ReAct is widely regarded as the foundational paradigm for agentic LLM systems -- it demonstrated that a single LLM, with only few-shot prompting and no task-specific training, could outperform imitation learning and reinforcement learning baselines trained on 10^5 task instances.

## Full Name

**ReAct (Reasoning + Acting)**

**Also Known As**: ReAct prompting, Thought-Action-Observation loop, interleaved reasoning and acting, ReAct agent

## Prompt Format

### Thought-Action-Observation Loop

ReAct augments the model's action space with a special `think` action. At each step, the model generates one of:

```
Thought[i]: Free-form reasoning about current state, plan, or information need
Action[i]:  Task-specific operation (e.g., Search[query], Lookup[term], Finish[answer])
Observation[i]: Environment feedback (returned by the external system, not generated by the model)
```

### Concrete Example (HotPotQA)

```
Question: What is the elevation range for the area that the eastern sector
          of the Colorado orogeny extends into?

Thought 1: I need to search Colorado orogeny, find the area that the eastern
           sector extends into, then find the elevation range of that area.
Action 1:  Search[Colorado orogeny]
Observation 1: The Colorado orogeny was an episode of mountain building...
               The eastern sector extends into the High Plains.

Thought 2: The eastern sector extends into the High Plains. I need to search
           High Plains and find its elevation range.
Action 2:  Search[High Plains]
Observation 2: High Plains refers to one of the two distinct land regions...
               elevation from around 1,800 to 7,000 ft.

Thought 3: High Plains rise in elevation from around 1,800 to 7,000 ft.
Action 3:  Finish[approximately 1,800 to 7,000 ft]
```

The key structural difference from [Chain of Thought](term_chain_of_thought.md) is that reasoning is interleaved with external actions rather than being a monolithic internal chain.

## Key Properties

| Property | Description |
|----------|-------------|
| **Interleaved** | Reasoning and acting alternate in a loop rather than being separate phases; each informs the other |
| **Grounded** | Actions retrieve external information, reducing hallucination compared to pure reasoning (6% vs. 14% false positive rate on HotPotQA) |
| **Training-free** | Works via few-shot prompting (1-6 examples) -- no fine-tuning, RL, or task-specific training required |
| **Task-general** | Same paradigm works across knowledge QA, fact verification, text games, and web navigation |
| **Interpretable** | Thought traces expose the model's reasoning process; actions expose its information-gathering decisions |
| **Human-editable** | Humans can inspect and edit Thought traces mid-trajectory to correct the agent's course |

## Empirical Results

### Knowledge-Intensive Tasks (PaLM-540B, few-shot)

| Task | Standard | CoT Only | Act Only | ReAct | ReAct + CoT-SC | Best |
|------|:--------:|:--------:|:--------:|:-----:|:--------------:|:----:|
| HotPotQA (EM, 6-shot) | 28.7 | 29.4 | 25.7 | 27.4 | **35.1** | ReAct+CoT |
| FEVER (Acc, 3-shot) | 57.1 | 56.3 | 58.9 | 60.9 | **64.6** | ReAct+CoT |

### Interactive Decision-Making Tasks (few-shot)

| Task | Act Only | ReAct | Supervised Baseline | Delta vs. Act |
|------|:--------:|:-----:|:-------------------:|:-------------:|
| ALFWorld (2-shot, success %) | 45 | **71** | 37 (trained on ~100K samples) | **+26** |
| WebShop (1-shot, success %) | 30.1 | **40** | 29.1 (trained on ~90K samples) | **+9.9** |

### Key Findings

- **ReAct vs. CoT**: On FEVER, ReAct outperforms CoT (60.9 vs. 56.3) because external retrieval grounds fact verification. On HotPotQA, CoT slightly leads (29.4 vs. 27.4) because some questions are answerable from parametric knowledge alone.
- **Hallucination reduction**: CoT's major failure mode is hallucination (56% of failures); ReAct's grounding in external observations reduces this substantially.
- **Best of both worlds**: Combining ReAct with CoT + Self-Consistency yields the highest scores on both tasks, leveraging internal reasoning when sufficient and external retrieval when needed.
- **Decision-making dominance**: On ALFWorld and WebShop, ReAct with just 1-2 examples outperforms RL/IL agents trained on ~100K samples by 34% and 10% absolute.

## Taxonomy: ReAct in the Reasoning Framework Landscape

| Framework | Internal Reasoning | External Actions | Interleaved | Key Innovation |
|-----------|:------------------:|:----------------:|:-----------:|----------------|
| Standard prompting | No | No | -- | Baseline |
| [Chain of Thought](term_chain_of_thought.md) | Yes | No | -- | Step-by-step reasoning traces |
| Act-only (WebGPT, SayCan) | No | Yes | -- | Tool use without reasoning |
| **ReAct** | **Yes** | **Yes** | **Yes** | **Synergized reasoning + acting** |
| Reflexion (Shinn 2023) | Yes | Yes | Yes | Self-reflection on failed trajectories |
| Tree of Thought (Yao 2023) | Yes | No | -- | Branching search over reasoning tree |
| Toolformer (Schick 2023) | Implicit | Yes | Implicit | Self-taught API calls via fine-tuning |

## Notable Systems and Implementations

- **LangChain**: Adopted ReAct as its default agent architecture; the `AgentExecutor` implements the Thought-Action-Observation loop
- **LangGraph**: Extends ReAct with stateful graph-based orchestration for multi-step agent workflows
- **Amazon Bedrock Agents**: Uses ReAct-style reasoning for orchestrating tool calls in managed agent services
- **Strands Agents SDK**: Amazon's open-source agent framework implementing ReAct-style action loops
- **AutoGPT / BabyAGI**: Early autonomous agent projects inspired by the ReAct interleaving paradigm
- **OpenAI Function Calling**: The tool-use API pattern can be seen as a productionized variant of ReAct's action mechanism

## Applications

### General Agentic AI
- **Foundation of language agents**: ReAct established the paradigm that LLMs can be controllers that reason about when and how to use tools, directly enabling the 2023-2024 explosion of agentic AI systems.
- **Tool-augmented LLMs**: The Action step in ReAct generalized to arbitrary tool calls (search, code execution, API calls), forming the conceptual basis for [function calling](term_function_calling.md).
- **Human-in-the-loop correction**: ReAct's interpretable Thought traces enable human operators to inspect, understand, and edit agent reasoning mid-trajectory.

### Applications to Our Work
- **[GreenTEA](term_greentea.md)** investigation automation uses ReAct-style interleaved reasoning and tool use in its agentic pipeline -- the agent reasons about which investigation step to take next, executes it, observes the result, and continues.
- **[Agent Orchestration](term_agent_orchestration.md)** patterns in our systems (LangGraph, Strands) are direct descendants of the ReAct framework, extended with state management and multi-agent coordination.
- **[Agent SOP](term_agent_sop.md)** enforcement benefits from ReAct's interpretable traces -- each Thought step can be audited against standard operating procedures.

## Challenges and Limitations

1. **Retrieval dependency**: ReAct's effectiveness is bounded by the quality of external retrieval. Poor search results cascade into flawed reasoning, causing error propagation across the trajectory.
2. **Structural rigidity**: The fixed Thought-Action-Observation format constrains reasoning flexibility compared to free-form CoT, sometimes leading to less fluid reasoning on knowledge-heavy tasks.
3. **Action space design**: Performance is sensitive to how the action space is defined. Too few actions limit capability; too many increase the search space and error rate.
4. **Latency overhead**: Each action step requires an external API call (search, lookup), adding wall-clock latency compared to pure CoT reasoning that stays within a single forward pass.
5. **Error compounding**: In multi-step trajectories, an early incorrect action can lead the agent down a wrong path. Unlike CoT + Self-Consistency, the base ReAct framework has no built-in mechanism for trajectory-level correction (addressed later by Reflexion).
6. **Human performance gap**: On both HotPotQA and WebShop, ReAct still lags significantly behind expert human performance, indicating substantial room for improvement.
