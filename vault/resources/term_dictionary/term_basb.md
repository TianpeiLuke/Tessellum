# Term: BASB (Building a Second Brain)

## Definition

**BASB** — **Building a Second Brain** — is a Personal Knowledge Management (PKM) methodology introduced by Tiago Forte in *Building a Second Brain: A Proven Method to Organize Your Digital Life and Unlock Your Creative Potential* (2022). The system externalizes cognition by capturing ideas digitally, organizing them for actionability, distilling them to their essence, and expressing them as creative output. Forte's central thesis extends David Allen's insight that "your mind is for having ideas, not holding them" from task management to *knowledge management*: the problem is not just unprocessed commitments (Allen's "open loops") but unprocessed *information* — the thousands of ideas, insights, and references that knowledge workers encounter daily and lose for lack of a reliable system.

## Full Name

**BASB** = **B**uilding **A** **S**econd **B**rain

Also known as:
- Second Brain methodology
- Forte's PKM system
- CODE method (the book's primary framework)

## Core Framework: CODE

The book's primary framework is the **CODE method** — a four-phase knowledge lifecycle:

| Phase | Action | Key Question | Technique |
|-------|--------|-------------|-----------|
| **Capture** | Gather information that resonates | "Does this resonate?" | Twelve Favorite Problems |
| **Organize** | Sort by actionability | "How will this help me move a current project forward?" | PARA framework |
| **Distill** | Extract the essence | "What is the most important point?" | Progressive Summarization |
| **Express** | Create and share output | "What can I create with this?" | Intermediate Packets |

## Key Techniques

| Technique | Phase | Description |
|-----------|-------|-------------|
| **Twelve Favorite Problems** | Capture | Feynman-inspired filter: maintain 12 open questions that matter most to you; capture anything that resonates with one |
| **PARA** | Organize | Four-category system sorting by actionability: Projects (with deadlines), Areas (ongoing), Resources (reference), Archives (inactive) |
| **Progressive Summarization** | Distill | Four layers of compression (Soil → Oil → Gold → Gems), each reducing by 80–90%; "a method for *forgetting* as much as possible" |
| **Intermediate Packets** | Express | Modular, reusable knowledge units (distilled notes, outtakes, WIP, deliverables, external docs) assembled into creative outputs |
| **Archipelago of Ideas** | Express | Overcome the blank page by gathering IPs into an outline before writing |
| **Hemingway Bridge** | Express | End sessions with notes for your future self to restart without friction |

| Dimension | GTD (Allen) | BASB (Forte) | Zettelkasten (Luhmann) | Atomic Habits (Clear) |
|-----------|------------|-------------|----------------------|----------------------|
| **Focus** | Task management | Knowledge management | Intellectual production | Behavior change |

## Vault Implementation

The vault directly implements several BASB concepts:

1. **C.O.D.E. skill taxonomy**: Each vault skill is tagged with a `code-stage` metadata field (capture, organize, distill, express) — directly derived from Forte's CODE method
2. **PARA-Zettelkasten hybrid**: Top-level structure (areas, resources, projects, archives, entry_points) mirrors PARA's categories, while individual notes follow Zettelkasten principles (atomic notes, bidirectional links, graph traversal)
3. **Notes as Intermediate Packets**: Term notes, paper section notes, digest notes, and skills are all modular IPs assembled into larger outputs via search and graph traversal
4. **Progressive Summarization in digests**: Overview sections serve as Layer 4 summaries; Key Takeaways as Layer 3 compression; full body as Layer 1–2
