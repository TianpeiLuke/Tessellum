# Generative Agents

## Definition

**Generative Agents** are computational software agents that simulate believable human behavior by extending a large language model with a novel cognitive architecture comprising a memory stream, reflection mechanism, and planning module. Introduced by Park et al. (2023) in "Generative Agents: Interactive Simulacra of Human Behavior," the system demonstrated that LLM-powered agents can produce coherent individual behavior and emergent social dynamics in a persistent, open-ended sandbox environment.

The foundational demonstration populated a virtual town called **Smallville** -- inspired by *The Sims* -- with 25 agents, each initialized with a brief natural-language description (name, occupation, relationships). From these seeds alone, agents autonomously wake up, cook breakfast, go to work, form opinions, initiate conversations, spread information, coordinate group activities, and form new relationships. Users can observe and intervene using natural language.

The work is widely regarded as the landmark paper that catalyzed the LLM agent research wave of 2023-2025, demonstrating that believable, persistent, socially interactive agents are achievable with careful architectural design around a foundation model.

## Historical Context

Believable agents have been a long-standing goal in artificial intelligence and human-computer interaction, with roots in:

- **The Sims** (2000) and earlier virtual life simulations that used hand-crafted finite state machines for NPC behavior
- **BDI (Belief-Desire-Intention) architectures** from multi-agent systems research (Bratman, 1987; Rao & Georgeff, 1995)
- **Cognitive architectures** such as SOAR and ACT-R that model human memory retrieval and planning
- **Interactive drama** research (Mateas & Stern, *Facade*, 2005) exploring believable character AI

Park et al. bridged these traditions with the capabilities of large language models, replacing hand-authored behavioral rules with LLM-driven generation conditioned on a structured memory system. The paper was published at **UIST '23** (36th Annual ACM Symposium on User Interface Software and Technology, October-November 2023, San Francisco) and used **GPT-3.5-turbo** as its backbone LLM (GPT-4 was invite-only at the time).

**Authors**: Joon Sung Park, Joseph C. O'Brien, Carrie J. Cai, Meredith Ringel Morris, Percy Liang, and Michael S. Bernstein (Stanford University / Google DeepMind).

## Architecture

The generative agent architecture extends an LLM with three core modules that form a perceive-retrieve-reflect-plan-act loop:

```
Environment Perception
        |
        v
  Memory Stream  <--- stores all observations as natural-language records
        |
        v
  Memory Retrieval --- scores by: recency + importance + relevance
        |
        v
  +-----+------+
  |            |
  v            v
Reflection   Planning
  |            |
  v            v
Higher-level   Hierarchical action plans
inferences     (day -> hour -> 5-15 min)
  |            |
  +-----+------+
        |
        v
    Agent Action / Dialogue
```

### Memory Stream

The **memory stream** is a comprehensive, append-only database recording the agent's experiences in natural language. Each memory object contains:

- **Description**: Natural-language text of the observation (e.g., "Isabella Rodriguez is setting out the pastries")
- **Creation timestamp**: When the observation occurred in the simulation
- **Last access timestamp**: When the memory was last retrieved

The memory stream captures both direct **observations** (what the agent perceives in the environment) and **reflections** (higher-level inferences generated by the agent itself). Both types are stored uniformly and retrieved through the same mechanism.

### Memory Retrieval

When an agent needs to act, the system retrieves relevant memories using a scoring function that combines three components:

**retrieval_score = alpha_recency * recency + alpha_importance * importance + alpha_relevance * relevance**

| Component | Mechanism | Intuition |
|-----------|-----------|-----------|
| **Recency** | Exponential decay based on time since last access | Recent experiences are more salient |
| **Importance** | LLM-assigned score (1-10) based on memory description | Mundane events (eating breakfast) score low; significant events (a breakup, a career change) score high |
| **Relevance** | Cosine similarity between embedding of memory text and embedding of current query/situation | Contextually related memories surface when needed |

All three scores are normalized to [0, 1] via min-max scaling. In the implementation, all alpha weights are set to 1, giving equal weight to each component.

### Reflection

**Reflection** is a secondary, higher-level memory type generated periodically by the agent. Reflections synthesize patterns across multiple observations into abstract inferences.

- **Trigger condition**: Reflections are generated when the sum of importance scores of recent (unreflected) events exceeds a threshold -- in practice, roughly **2-3 times per simulated day**
- **Focal point generation**: The system takes the 100 most recent memories and asks the LLM: *"Given only the information above, what are the 3 most salient high-level questions we can answer about the subjects in the statements?"*
- **Insight generation**: For each focal question, the system retrieves relevant memories and asks the LLM to generate high-level insights with **evidence pointers** (e.g., "Klaus Mueller is dedicated to his research on gentrification (because of 1, 2, 8, 15)")
- **Storage**: Reflections are stored back into the memory stream as first-class memory objects, enabling reflections-on-reflections -- building a tree of increasingly abstract understanding

### Planning

The planning module generates agent behavior through **top-down recursive decomposition**:

1. **Day plan**: Based on agent description and previous day's summary, the LLM generates a broad plan with 5-8 high-level activities
2. **Hourly decomposition**: Each high-level activity is broken into hour-level chunks
3. **Fine-grained actions**: Hour-level chunks are decomposed into 5-15 minute action sequences

Plans are not rigid. Agents continuously perceive their environment, and the LLM decides whether to **maintain the current plan** or **react** to new observations. When an agent encounters another agent, the system generates natural dialogue by conditioning on both agents' memories of each other.

## Key Properties

- **Believability**: In controlled evaluations, human raters judged generative agents as more believable than ablated versions and human-authored baselines
- **Emergent social behavior**: Complex social phenomena arise without explicit programming:
  - Information diffusion (Valentine's Day party knowledge spread from 4% to 48% of agents over two simulated days)
  - Relationship formation (social network density increased over time)
  - Coordination (5 of 12 invited agents attended the party; others gave realistic excuses)
- **Ablation sensitivity**: Each architectural component (observation, planning, reflection) contributes critically -- removing any one degrades believability significantly. Removing reflection had the most pronounced negative effect
- **Natural-language grounding**: All reasoning, memory, and planning operate in natural language, making the system interpretable and debuggable
- **Persistent identity**: Agents maintain consistent personality, relationships, and life narratives over extended simulation periods

## Applications

- **Social science research**: Simulating social dynamics, opinion formation, and collective behavior without human subjects
- **Game design**: NPCs with persistent memory and emergent social relationships
- **Urban planning and policy modeling**: Agent-based simulations of community responses to interventions
- **Training and education**: Interactive scenarios with believable characters
- **Human-AI interaction design**: Understanding how users relate to persistent, memory-capable agents
- **Abuse detection relevance**: The architecture demonstrates how adversarial agents could simulate coordinated social manipulation, and how memory-based behavioral profiling can detect anomalous patterns in agent-driven systems

## Challenges

- **Computational cost**: Each agent requires frequent LLM calls for perception, retrieval, reflection, and planning -- scaling to hundreds or thousands of agents is expensive
- **Hallucination and confabulation**: Agents may generate memories or reflections inconsistent with their actual experiences
- **Evaluation difficulty**: Measuring "believability" is inherently subjective; the paper relies on human evaluation which is costly and non-reproducible
- **Memory scalability**: The memory stream grows unboundedly; retrieval over very long histories may degrade in quality
- **Limited grounding**: Agents have no embodied perception -- all environmental information is mediated through text descriptions
- **Temporal coherence**: Over very long simulations, personality drift and logical inconsistencies can accumulate
- **Behavioral diversity**: All agents share the same LLM backbone, which may limit the range of genuinely distinct personalities

## Related Terms

- **[Emergence](term_emergence.md)** -- The social behaviors in Smallville (party coordination, information diffusion) are canonical examples of weak emergence in agent systems
- **[Systems Thinking](term_systems_thinking.md)** -- Generative agents demonstrate systems-level behavior arising from individual agent interactions and feedback loops
- **[Agentic Memory](term_agentic_memory.md)** -- The memory stream architecture is a precursor to agentic memory systems like A-MEM; both emphasize agent-driven memory organization, though A-MEM adds self-organizing link generation
- **[LLM](term_llm.md)** -- The foundational technology enabling generative agents; the architecture wraps an LLM with structured memory and planning scaffolding
- **[Multi-Agent Collaboration](term_multi_agent_collaboration.md)** -- Generative agents demonstrate emergent collaboration without explicit coordination protocols, contrasting with engineered multi-agent frameworks like AutoGen
- **[Agent Orchestration](term_agent_orchestration.md)** -- Unlike orchestrated multi-agent systems with deterministic control flow, generative agents coordinate autonomously through environmental perception and social interaction
- **[Self-Evolving Agent](term_self_evolving_agent.md)** -- The reflection mechanism is an early form of self-improvement through experience synthesis
